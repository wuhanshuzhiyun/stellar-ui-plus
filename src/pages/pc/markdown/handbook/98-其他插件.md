# 内部其他插件

## 1. ste-vue-inset-loader

### 简介

编译阶段在sfc模板指定位置插入自定义内容，适用于webpack构建的vue应用，常用于小程序需要全局引入组件的场景。

由于小程序没有开放根标签，没有办法在根标签下追加全局标签，所以要使用组件必须在当前页面引入组件标签。

### 特性

- 支持h5，app等多端。
- 编译时自动注入自定义内容
- 支持全局和页面级配置
- 灵活的标签配置方式

### 安装

```bash
    npm install ste-vue-inset-loader --save-dev
```

### 配置

#### 在 vue.config.js 中注入 loader

```javascript
const path = require('path');
module.exports = {
    chainWebpack: config => {
        config.module
            .rule('vue')
            .test(/\.vue$/)
            .use()
            .loader(path.resolve(__dirname, './node_modules/ste-vue-inset-loader'))
            .end();
    },
};
```

#### 在 pages.json 中配置 insetLoader

```json
{
    "insetLoader": {
        "config": {
            "confirm": "<BaseConfirm ref='confirm'></BaseConfirm>",
            "abc": "<BaseAbc ref='BaseAbc'></BaseAbc>"
        },
        // 全局配置
        "label": ["confirm"],
        "rootEle": "view"
    },
    "pages": [
        {
            "path": "pages/tabbar/index/index",
            "style": {
                "navigationBarTitleText": "测试页面",
                // 单独配置，用法跟全局配置一致，优先级高于全局
                "label": ["confirm", "abc"],
                "rootEle": "view"
            }
        }
    ]
}
```

### 配置说明

| 参数      | 类型     | 默认值 | 说明                                                                            |
| --------- | -------- | ------ | ------------------------------------------------------------------------------- |
| `config`  | `Object` | `{}`   | 定义标签名称和内容的键值对                                                      |
| `label`   | `Array`  | `[]`   | 需要全局引入的标签，打包后会在所有页面引入此标签                                |
| `rootEle` | `String` | ``     | 目标元素的标签类型，没有时直接放在元素上面（存在rootEle属性时则挂载到该元素中） |

> **注意**：`label` 和 `rootEle` 支持在单独页面的 style 里配置，优先级高于全局配置。

### 常见问题

**问题**：运行时出现 `Unexpected token } in JSON at position 256` 错误

**解决**：请删除 `pages.json` 文件中多余的空格

## 2. ste-helper

### 简介

StellarUI-Plus 组件库的 VS Code 插件，旨在帮助开发者更加高效地使用 StellarUI-Plus 来开发项目。

### 版本信息

- **当前版本**：0.0.202
- **发布者**：StellarUI
- **VS Code 要求**：^1.56.0

### 主要特性

#### 1. 快速查看文档

移动鼠标到 StellarUI-Plus 的标签上能够看到一行查看当前组件文档的提示，移到属性或方法，会给出属性和方法的解释，点击可以跳转到对应组件的文档，方便开发者查阅。

#### 2. 代码自动补全

在输入 `ste-` 时，会出现组件列表提示，这时，可选择任意需要的组件，然后点击回车自动补全，省时又省力～

#### 3. 属性和方法提示

组件中输入一个空格会提示该组件所有的属性和方法的说明。

### 使用方式

#### 安装插件

在 VS Code 扩展市场搜索 `ste-helper` 或访问插件地址进行安装。

#### 激活插件

插件会在以下语言环境中自动激活：

- Vue
- JavaScript
- TypeScript

### 技术栈

- TypeScript
- VS Code Extension API

### 开发脚本

```bash
# 生成组件映射
pnpm run gen

# 编译插件
pnpm run compile

# 监听文件变化
pnpm run watch

# 打包插件
pnpm run package

# 发布插件
pnpm run publish
```

### 更新日志

插件会随着 StellarUI-Plus 组件库的更新而持续优化，新增组件时会自动更新代码提示。

## 3. ste-cli

### 简介

Stellar 配套的脚手架工具，帮助开发者快速创建和管理项目。

### 特性

- 支持创建 Vue2 和 Vue3 模板
- 快速项目初始化
- 统一的项目结构
- 内置最佳实践配置

### 使用方式

#### 安装

```bash
npm install -g ste-cli
```

#### 创建项目

```bash
# 创建 Vue2 项目
ste-cli create my-project --template vue2

# 创建 Vue3 项目
ste-cli create my-project --template vue3
```

### 模板说明

| 模板类型  | 描述                       | 适用场景                 |
| --------- | -------------------------- | ------------------------ |
| Vue2 模板 | 基于 Vue 2 的 uni-app 项目 | 需要兼容旧版本项目       |
| Vue3 模板 | 基于 Vue 3 的 uni-app 项目 | 新项目开发，使用最新特性 |

### 项目结构

使用脚手架创建的项目包含以下内容：

- 完整的目录结构
- 配置文件（vite.config.js、package.json 等）
- 示例代码和组件
- 开发和构建脚本

## 4. ste-camera

### 简介

适用 App 端相机插件，适用于 uni-app，提供原生相机功能支持。

### 特性

- 原生相机调用
- 支持多种拍摄模式
- 图片和视频拍摄
- 前后摄像头切换
- 闪光灯控制
- 对焦功能

### 使用场景

- 用户头像上传
- 身份证拍摄
- 扫码识别
- 视频录制
- 实时预览

### 安装

#### HBuilderX 方式

1. 在 HBuilderX 中，点击 `插件市场`
2. 搜索 `ste-camera`
3. 点击 `导入` 或 `使用`

#### 手动安装

```bash
npm install ste-camera
```

### 基本使用

```vue
<template>
    <view>
        <ste-camera @takePhoto="handleTakePhoto" @recordVideo="handleRecordVideo" />
    </view>
</template>

<script setup>
const handleTakePhoto = photo => {
    console.log('拍摄的照片', photo);
};

const handleRecordVideo = video => {
    console.log('录制的视频', video);
};
</script>
```

### API 说明

| 事件名      | 说明               | 回调参数                             |
| ----------- | ------------------ | ------------------------------------ |
| takePhoto   | 拍照完成时触发     | `{ path: string, size: number }`     |
| recordVideo | 录制视频完成时触发 | `{ path: string, duration: number }` |
| error       | 发生错误时触发     | `{ message: string, code: number }`  |

## 5. ste-canvas-poster

### 简介

基于 Canvas 2D API 的声明式海报绘制引擎，支持微信小程序与 APP 双端。通过 JSON Schema 描述海报结构，一套代码同时适配多端。

### 特性

- **Schema 驱动** — 通过 JSON 描述海报结构，数值直接使用设计稿 rpx
- **双端一致** — 一套 Schema 同时适配微信小程序与 APP，无需条件编译
- **模板变量** — Schema 中 `{{key}}` 自动替换为 data 数据
- **Flex 布局** — view 容器支持 `display: 'flex'` 及子元素 margin、baseline 对齐
- **自动高度** — text 元素可省略 height，引擎根据内容自动计算
- **内置二维码** — qrcode 类型直接生成二维码，无需额外依赖
- **TypeScript 支持** — 完整类型声明，开发时自动补全

### 安装

```bash
npm install ste-canvas-poster
```

### 基本使用

#### 1. 定义 Schema

```javascript
const schema = {
    width: 750,
    height: 1068,
    background: '#FFFFFF',
    views: [
        {
            type: 'view',
            css: {
                left: 54,
                top: 54,
                width: 642,
                height: 960,
                borderRadius: 20,
                background: '#FFFFFF',
            },
        },
        {
            type: 'image',
            src: '{{coverImage}}',
            css: {
                left: 96,
                top: 160,
                width: 560,
                height: 478,
                borderRadius: 12,
                objectFit: 'cover',
            },
        },
        {
            type: 'text',
            text: '{{titleText}}',
            css: {
                left: 96,
                top: 730,
                fontSize: 32,
                fontWeight: 'bold',
                color: '#181818',
                maxWidth: 560,
                ellipsis: true,
            },
        },
        {
            type: 'qrcode',
            src: '{{qrcodeUrl}}',
            css: { left: 504, top: 842, width: 160, height: 160 },
        },
    ],
};
```

#### 2. 准备数据

```javascript
const data = {
    coverImage: 'https://example.com/cover.jpg',
    titleText: '精选商品',
    qrcodeUrl: 'https://example.com',
};
```

#### 3. 渲染与保存

```javascript
import { renderPoster } from 'ste-canvas-poster';

const engine = await renderPoster({
    schema,
    data,
    selector: '#myCanvas',
    vm: this,
});

await engine.saveToAlbum();
const tempPath = await engine.toTempFilePath();
```

#### 4. Template 模板

```vue
<template>
    <!-- #ifdef MP-WEIXIN -->
    <canvas id="myCanvas" type="2d" class="canvas" :style="canvasStyle" />
    <!-- #endif -->
    <!-- #ifdef APP-PLUS -->
    <canvas canvas-id="myCanvas" id="myCanvas" class="canvas" :style="canvasStyle" />
    <!-- #endif -->
</template>
```

```javascript
computed: {
    canvasStyle() {
        return { width: '750rpx', height: '1068rpx' };
    }
}
```

### API 说明

#### renderPoster(options)

渲染海报，自动处理 rpx 转换和平台差异。

| 参数       | 类型                  | 必填 | 默认值   | 说明                                  |
| ---------- | --------------------- | ---- | -------- | ------------------------------------- |
| `schema`   | `PosterSchema`        | 是   | -        | 海报结构描述                          |
| `data`     | `TemplateData`        | 否   | `{}`     | 模板变量数据                          |
| `selector` | `string`              | 是   | -        | Canvas 选择器，如 `'#myCanvas'`       |
| `vm`       | `Record<string, any>` | 是   | -        | Vue 组件实例                          |
| `dpr`      | `number`              | 否   | 自动获取 | 像素比                                |
| `useRpx`   | `boolean`             | 否   | `true`   | 是否将 schema 数值视为 rpx 并自动转换 |

**返回值**：`Promise<PosterEngine>` — 引擎实例

#### PosterEngine 方法

| 方法                       | 说明                             | 返回值                           |
| -------------------------- | -------------------------------- | -------------------------------- |
| `saveToAlbum()`            | 导出图片并保存到系统相册         | `Promise<string>` — 临时文件路径 |
| `toTempFilePath(options?)` | 导出为临时文件路径，不保存到相册 | `Promise<string>` — 临时文件路径 |
| `destroy()`                | 销毁引擎实例，释放图片缓存等资源 | `void`                           |

### Schema 元素类型

| 类型     | 说明                                   | 主要属性                                                                                                          |
| -------- | -------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `view`   | 容器，支持背景、边框、圆角和 Flex 布局 | `background`、`borderWidth`、`borderColor`、`display`、`flexDirection`、`alignItems`、`justifyContent`、`padding` |
| `image`  | 图片，支持圆角裁剪和 objectFit         | `src`、`objectFit`（fill/cover/contain/widthFix/heightFix）                                                       |
| `text`   | 文本，支持多行换行、省略号、删除线     | `text`、`fontSize`、`fontWeight`、`color`、`textAlign`、`lineHeight`、`maxWidth`、`ellipsis`、`lines`             |
| `qrcode` | 二维码，内容支持模板变量               | `text`、`color`、`background`                                                                                     |

### CSS 属性参考

#### 通用属性

| 属性           | 类型                 | 默认值 | 说明                          |
| -------------- | -------------------- | ------ | ----------------------------- |
| `left`         | `number`             | `0`    | x 坐标                        |
| `top`          | `number`             | `0`    | y 坐标                        |
| `width`        | `number`             | -      | 宽度                          |
| `height`       | `number`             | -      | 高度（text 可省略）           |
| `opacity`      | `number`             | `1`    | 透明度（0-1）                 |
| `borderRadius` | `number \| number[]` | `0`    | 圆角，支持 `[lt, rt, rb, lb]` |

#### text 属性

| 属性         | 类型                            | 默认值      | 说明       |
| ------------ | ------------------------------- | ----------- | ---------- |
| `fontSize`   | `number`                        | `14`        | 字号       |
| `fontWeight` | `string \| number`              | `'normal'`  | 字重       |
| `color`      | `string`                        | `'#000000'` | 文本颜色   |
| `textAlign`  | `'left' \| 'center' \| 'right'` | `'left'`    | 对齐方式   |
| `lineHeight` | `number`                        | `1.4`       | 行高倍数   |
| `maxWidth`   | `number`                        | -           | 最大宽度   |
| `ellipsis`   | `boolean`                       | `false`     | 单行省略号 |
| `lines`      | `number`                        | `0`         | 最大行数   |

### 命名规范

所有 Schema 及 CSS 属性统一使用 **camelCase**（驼峰命名）。
