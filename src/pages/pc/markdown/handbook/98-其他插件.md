# 内部其他插件

## 1. vue3-inset-loader

### 简介
编译阶段在 SFC 模板指定位置插入自定义内容，适用于 Vite 构建的 Vue 应用，常用于小程序需要全局引入组件的场景。

由于小程序没有开放根标签，没有办法在根标签下追加全局标签，所以要使用组件必须在当前页面引入组件标签。

### 特性
- 支持 Vue3 + Vite 构建
- 编译时自动注入自定义内容
- 支持全局和页面级配置
- 灵活的标签配置方式

### 安装

#### 第一步：放置文件
将 `vue3-inset-loader` 文件夹放置在项目的 `plugins/` 目录下。

#### 第二步：安装依赖
```bash
pnpm i strip-json-comments@3.1.1 @vue/compiler-sfc -D
```

### 配置

#### 在 vite.config.js 中注入 loader
```javascript
import { defineConfig } from 'vite'
import vue3InsetLoaderPlugin from './plugins/vue3-inset-loader'

export default defineConfig({
  build: {
    sourcemap: process.env.NODE_ENV === 'development',
  },
  plugins: [
    // 注入loader，放在plugins的首位
    vue3InsetLoaderPlugin
  ],
})
```

#### 在 pages.json 中配置 insetLoader
```json
{
  "insetLoader": {
    "config": {
      "confirm": "<BaseConfirm ref='confirm'></BaseConfirm>",
      "abc": "<BaseAbc ref='BaseAbc'></BaseAbc>"
    },
    // 全局配置
    "label": ["confirm"],
    "rootEle": "view"
  },
  "pages": [
    {
      "path": "pages/tabbar/index/index",
      "style": {
        "navigationBarTitleText": "测试页面",
        // 单独配置，用法跟全局配置一致，优先级高于全局
        "label": ["confirm", "abc"],
        "rootEle": "view"
      }
    }
  ]
}
```

### 配置说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|---------|------|
| `config` | `Object` | `{}` | 定义标签名称和内容的键值对 |
| `label` | `Array` | `[]` | 需要全局引入的标签，打包后会在所有页面引入此标签 |
| `rootEle` | `String` | `` | 目标元素的标签类型，没有时直接放在元素上面（存在rootEle属性时则挂载到该元素中） |

> **注意**：`label` 和 `rootEle` 支持在单独页面的 style 里配置，优先级高于全局配置。

### 常见问题

**问题**：运行时出现 `Unexpected token } in JSON at position 256` 错误

**解决**：请删除 `pages.json` 文件中多余的空格

## 2. ste-helper

### 简介
StellarUI-Plus 组件库的 VS Code 插件，旨在帮助开发者更加高效地使用 StellarUI-Plus 来开发项目。

### 版本信息
- **当前版本**：0.0.202
- **发布者**：StellarUI
- **VS Code 要求**：^1.56.0

### 主要特性

#### 1. 快速查看文档
移动鼠标到 StellarUI-Plus 的标签上能够看到一行查看当前组件文档的提示，移到属性或方法，会给出属性和方法的解释，点击可以跳转到对应组件的文档，方便开发者查阅。

#### 2. 代码自动补全
在输入 `ste-` 时，会出现组件列表提示，这时，可选择任意需要的组件，然后点击回车自动补全，省时又省力～

#### 3. 属性和方法提示
组件中输入一个空格会提示该组件所有的属性和方法的说明。

### 使用方式

#### 安装插件
在 VS Code 扩展市场搜索 `ste-helper` 或访问插件地址进行安装。

#### 激活插件
插件会在以下语言环境中自动激活：
- Vue
- JavaScript
- TypeScript

### 技术栈
- TypeScript
- VS Code Extension API

### 开发脚本
```bash
# 生成组件映射
pnpm run gen

# 编译插件
pnpm run compile

# 监听文件变化
pnpm run watch

# 打包插件
pnpm run package

# 发布插件
pnpm run publish
```

### 更新日志
插件会随着 StellarUI-Plus 组件库的更新而持续优化，新增组件时会自动更新代码提示。

## 3. ste-cli

### 简介
Stellar 配套的脚手架工具，帮助开发者快速创建和管理项目。

### 特性
- 支持创建 Vue2 和 Vue3 模板
- 快速项目初始化
- 统一的项目结构
- 内置最佳实践配置

### 使用方式

#### 安装
```bash
npm install -g ste-cli
```

#### 创建项目
```bash
# 创建 Vue2 项目
ste-cli create my-project --template vue2

# 创建 Vue3 项目
ste-cli create my-project --template vue3
```

### 模板说明

| 模板类型 | 描述 | 适用场景 |
|----------|---------|----------|
| Vue2 模板 | 基于 Vue 2 的 uni-app 项目 | 需要兼容旧版本项目 |
| Vue3 模板 | 基于 Vue 3 的 uni-app 项目 | 新项目开发，使用最新特性 |

### 项目结构
使用脚手架创建的项目包含以下内容：
- 完整的目录结构
- 配置文件（vite.config.js、package.json 等）
- 示例代码和组件
- 开发和构建脚本

## 4. ste-camera

### 简介
适用 App 端相机插件，适用于 uni-app，提供原生相机功能支持。

### 特性
- 原生相机调用
- 支持多种拍摄模式
- 图片和视频拍摄
- 前后摄像头切换
- 闪光灯控制
- 对焦功能

### 使用场景
- 用户头像上传
- 身份证拍摄
- 扫码识别
- 视频录制
- 实时预览

### 安装

#### HBuilderX 方式
1. 在 HBuilderX 中，点击 `插件市场`
2. 搜索 `ste-camera`
3. 点击 `导入` 或 `使用`

#### 手动安装
```bash
npm install ste-camera
```

### 基本使用
```vue
<template>
  <view>
    <ste-camera
      @takePhoto="handleTakePhoto"
      @recordVideo="handleRecordVideo"
    />
  </view>
</template>

<script setup>
const handleTakePhoto = (photo) => {
  console.log('拍摄的照片', photo);
};

const handleRecordVideo = (video) => {
  console.log('录制的视频', video);
};
</script>
```

### API 说明

| 事件名 | 说明 | 回调参数 |
|--------|------|----------|
| takePhoto | 拍照完成时触发 | `{ path: string, size: number }` |
| recordVideo | 录制视频完成时触发 | `{ path: string, duration: number }` |
| error | 发生错误时触发 | `{ message: string, code: number }` |
